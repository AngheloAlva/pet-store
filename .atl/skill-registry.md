## Compact Rules (injected into sub-agents)

### typescript
- Prefer `type` aliases over `interface` unless declaration merging is needed.
- Use `satisfies` for config literals to keep inferred literal types.
- No `any`. Use `unknown` and narrow.
- Enable strict mode (`strict: true`) — project tsconfig already does.

### tailwind-4
- Use `cn()` from `@/lib/utils` to merge classes.
- Theme tokens via `--color-*` and `--font-*` CSS vars defined in `globals.css` (`@theme inline`). Do NOT use `var(--...)` inside `className`; use the Tailwind token names that the `@theme` block creates.
- `@apply` only in `globals.css` base layer. Prefer utilities in JSX.

### zustand-5
- Use `create()` with `persist` middleware for cart-like stores.
- Export selectors as free functions (`selectTotalItems`) — enables structural sharing.
- Guard hydration-dependent UI with a `mounted` flag when reading persisted state.

### Base UI (project-specific)
- `<Button render={<Link href="/foo" />}>Text</Button>` — NEVER `asChild`.
- `<SheetTrigger render={<Button variant="ghost" />}>...</SheetTrigger>`.
- The render prop accepts `ReactElement | ((props, state) => ReactElement)`.

## How to Use

Orchestrator reads this file (and/or the engram `skill-registry` observation) ONCE per session. For each delegation, it matches the sub-agent's task to relevant skills and injects the matching **Compact Rules** blocks into the sub-agent prompt as `## Project Standards (auto-resolved)`.

Sub-agents do NOT read SKILL.md files. They receive pre-digested rules.
